Case study · Redesigning an AI-first tool, from scratch

Redesigning
Presentations.ai

The world's best AI presentation maker!

  • Role Product design
  • Where Presentations.ai
  • Shipped 24 Feb 2026
  • Scope Architecture, dashboard, invites

The world's most used business tool hasn't changed in decades. So we rebuilt ours from scratch.

The gap we exist to close

Presentations are the language of business, yet the software used to make them hasn't fundamentally changed since the 1990s. PowerPoint and Google Slides digitised paper presentations. They never transformed how we create them.

Presentations.ai exists to close the gap between a raw idea and a finished, professional deck. Not eventually. Not after hours of manual work. Right now. Instantly.

The product was holding itself back

There was no shortage of ideas for where the product could go. The existing architecture made most of them impossible to ship. New markets, core features, industry standards; none of it could happen on top of what existed. So we set out to recreate the foundation: scalable, backend driven, and a significantly better experience to use.

This case study is thin on metrics. The product was too broken for data to lead; we went back to first principles.

The company's event tracking and funnels could not give granular insights yet. So this is about the thinking, the decisions, and the work. Numbers will come as the product matures. If you want to see how I work with data, read the Hotstar case study.

Three fronts, one rebuild

ArchitectureFile organisation, collaboration, invite flows and granular access control. Enterprise ready.
ExperienceCore flows rethought, navigation simplified, key actions surfaced. Brought up to the standard users expect from modern tools.

Every presentation lived in one endless pile

No folders, no nesting, no grouping. Every deck sat in the same flat list regardless of project, client, or team. There was no way to organise, sort, or find your work efficiently. The product punished its most active users.

Built for solo use; work isn't

Nobody makes a presentation just for themselves. It gets shared, reviewed, edited, handed off. The product assumed a single user working alone. Real work doesn't look like that.

The new file architecture

Everything splits into two spaces: personal and team. Personal space is a simple filtered view of your work; no folders, no hierarchy. Team space is structured; every file sits in a folder, public or private. Permissions follow location. New and uncategorised files rest in drafts until they move to a project.

  • 1All personal content is private by default
  • 2Personal space has no folder structure
  • 3Content shared with you lives in Shared with me
  • 4No nested folders anywhere in the system
  • 5Team folders default to public visibility
  • 6File permissions always inherit from the parent folder

The dashboard

Everything starts here. Create, find, organise and collaborate from one screen.

  • Plan visibility at a glance so users always know what they're on
  • Credits always visible so users know where they stand
  • Profile and workspace settings separated for clarity
  • Global search always within reach
  • Entry points that guide you straight into the create flow
  • Relevant features surfaced where you need them, not buried in menus
  • Visual cues that split public folders from private ones
  • Larger thumbnails so you spot your work without reading titles
  • Shared with me as a dedicated space
  • Metadata on every file: where it lives and who made it

Rethinking invites

Sharing isn't one action; it's three. Workspace level, project level, file level. Each behaves differently depending on plan, role, and whether you're an admin. A free user inviting a teammate sees a different flow than a team admin with open seats. A file in a public project follows different rules than a file in drafts. We mapped every combination across four plan types, multiple permission roles and file locations, then designed flows that absorb the complexity instead of exposing it. For the complete invite logic across plans, roles and edge cases, read the full share-flow documentation.

  • 1Workspace invites require available seats
  • 2Non-admins can invite; an admin approves
  • 3Public projects open to every workspace member
  • 4Private projects require explicit invites
  • 5File sharing adapts to where the file lives
  • 6Guests get instant access; full seats need admin approval

Link sharing got the same treatment:

  • Copy a link with permissions baked in; the link changes with the access level
  • Toggle permission and access for any guest or member
  • Control who can access via link, and what they can do, from one settings view

The redesign cleared the way. Now we build forward

The redesign touched every layer of the product. Architecture, dashboard, invite flows, permissions, file management; all rebuilt from the ground up. It shipped on 24 February 2026 with about 80 percent of the redesign live and the rest rolling out. Too early to talk numbers, but the bottleneck that held everything back is gone. New features are already being built on top of what didn't exist a few months ago.

Helping Presentations.ai build better presentations

Next up: refresh. Paste or upload your data and let the AI handle the rest, then review what changed on every slide. Keep what works, adjust what doesn't.

Right now my time goes into better slide generation. That means understanding the create flow at a deeper level: how slides are structured, what objects and nodes exist, how content gets generated. The model builds an outline first, then generates slides from it. I curate input datasets and fine tune each stage so the output is better by default. The bigger goal: slides that aren't bound by templates at all. Freeform, generated from scratch, every time.
